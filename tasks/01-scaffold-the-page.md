# Task 1: Scaffold the Page

In this step, we created the basic file structure for the kanban app:

- `index.html`
- `styles.css`
- `script.js`

The goal was to get a working page on screen before adding any interactivity.

## What was added

In `index.html`, we set up:

- the page title
- a header with the app name
- a board area
- links to the CSS and JavaScript files

We also included one sample column and one sample task so the layout could be seen immediately.

In `styles.css`, we added:

- page background styling
- a centered app container
- card-like column styling
- simple task item styling

In `script.js`, we added a placeholder `console.log()` so the JavaScript file was connected and ready for the next step.

## Why this step matters

This gives us a clean starting point:

- the files are separated by responsibility
- the UI already looks like a kanban board
- the next tasks can focus only on behavior

## Result

After Task 1, opening `index.html` shows a minimal static kanban board with one column and one item.
